<span data-ttu-id="38322-105">O linie de ofertă bazată pe proiecte are conceptul de componente *incluse* și componente *taxabile*.</span><span class="sxs-lookup"><span data-stu-id="38322-105">A project-based quote line has the concept of *included* components and *chargeable* components.</span></span>

<span data-ttu-id="38322-106">Componentele incluse sunt supuse la:</span><span class="sxs-lookup"><span data-stu-id="38322-106">Included components are subject to:</span></span>

  - <span data-ttu-id="38322-109">Setările frecvenței facturilor definite pe linia de ofertă bazată pe proiect</span><span class="sxs-lookup"><span data-stu-id="38322-109">Invoice frequency settings defined on the project-based quote line</span></span>

<span data-ttu-id="38322-110">Un subset al componentelor incluse poate fi marcat ca taxabil folosind câmpul **Tip de facturare**.</span><span class="sxs-lookup"><span data-stu-id="38322-110">A subset of the included components can be marked as chargeable using the **Billing Type** field.</span></span> <span data-ttu-id="38322-111">Câmpul **Tip de facturare** este un set de opțiuni care permite următoarele valori în contextul unei linii de ofertă:</span><span class="sxs-lookup"><span data-stu-id="38322-111">The **Billing Type** field is an option-set that allows the following values in the context of a quote line:</span></span>

  - <span data-ttu-id="38322-113">Netaxabil</span><span class="sxs-lookup"><span data-stu-id="38322-113">Non-chargeable</span></span>

<span data-ttu-id="38322-114">Componentele taxabile pot fi definite pe sarcini, roluri și categorii de tranzacții.</span><span class="sxs-lookup"><span data-stu-id="38322-114">Chargeable components can be defined on tasks, roles, and transaction categories.</span></span>

<span data-ttu-id="38322-122">O sarcină de proiect poate fi taxabilă sau netaxabilă în contextul unei linii specifice de ofertă bazate pe proiect, ceea ce face posibilă următoarea configurare:</span><span class="sxs-lookup"><span data-stu-id="38322-122">A project task can be chargeable or non-chargeable in the context of a specific project-based quote line, which makes the following setup possible:</span></span>
